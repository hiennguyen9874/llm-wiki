# MoonDev

## Summary
MoonDev is referenced in the trading transcripts as a data and tooling ecosystem for liquidation and market data. The sources treat it as part of the infrastructure used to explore trading bots, liquidation-based strategies, and backtest experiments.

The newest source also links MoonDev to a Polymarket backtest workflow that uses `mundev.com/data` to get 1-minute data and tick-level data for short-interval bot experiments.

## Key points
- Appears as a recurring reference across multiple trading sources.
- Provides liquidation and market data used in strategy research.
- Also appears to include an app / docs / API surface that supports bot experimentation.
- Is tied to backtesting workflows rather than discretionary trading commentary.
- Appears in the new OpenClaw source as the data source for BTC and 1h / 6h datasets.
- Appears again in the Polymarket 5-minute bot transcript as a 1-minute / tick-data source.
- The spelling varies between `MoonDev` and `Mundave` in the transcripts.

## Relationships
- `MoonDev` provides liquidation and market data to trading workflows.
- `MoonDev` supports backtest and agent-based strategy experimentation.
- `MoonDev` is related to the trading-research threads around Claude Code, GPT-5.5, Claude Opus 4.7, and Polymarket.
- `MoonDev` may function as both a data provider and an app / API layer.

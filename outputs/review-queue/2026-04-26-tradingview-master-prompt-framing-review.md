# Review - TradingView master prompt framing

## Decision needed
Should the video’s “one prompt / under 4 minutes / no terminal” framing be treated as a durable topic-level UX claim on [[tradingview-mcp]], or stay mostly source-local until it is reinforced by another independent source?

## Recommendation
Keep it as a low-confidence demo claim on the topic page for now, and avoid turning it into a separate synthesis page unless more sources repeat the same setup story.

## Context
The source presents Claude + TradingView as a mostly no-code workflow:
- paste a master prompt
- confirm a couple setup steps
- have the bridge clone/configure itself
- use the resulting command surface to read charts and generate indicators

That framing is useful, but it is also promotional and may hide real setup complexity.

## Options
1. Keep the master-prompt framing as a topic-level demo claim only.
2. Create a separate prompt/setup synthesis page later if more sources repeat the pattern.
3. Down-rank the claim to source-local notes only.
